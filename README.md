# Incident Response and Security Breach Simulation

This repository contains documentation for Task 16: Incident Response & Security Breach Simulation.

## Objective
- Simulate a security incident
- Analyze logs for suspicious activity
- Classify and contain the incident
- Remove the threat
- Restore system to secure state
- Document incident timeline
- Recommend preventive measures

## Tools Used
- Linux Logs
- Windows Event Viewer
- TheHive (Alternative)

## Task Overview
A simulated security incident was created in a lab environment. Logs were analyzed to detect suspicious activity. The incident was classified, contained, eradicated, and documented properly.

## Key Learnings
- Incident detection techniques
- Log analysis skills
- Containment and recovery strategies
- Importance of documentation

## Outcome
This task improved practical understanding of incident response and breach management processes.
